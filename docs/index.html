Core Capabilities

Everything your app needs,
nothing you don't

Six core capabilities, built to work together from day one.

💬
Chat & Channels

Real-time messaging with channels, threads, reactions, read receipts, and file sharing. Backed by Kafka event streaming for zero message loss.

→
📹
HD Video Calls

1-on-1 and group calls powered by mediasoup WebRTC SFU. Adaptive bitrate, screen sharing, and recording — all WebRTC-native, not peer-to-peer.

→
🎙
Group Meetings

Schedule, join, and host meetings with participant management, raise-hand controls, mute-all, and full session analytics out of the box.

→
📡
Live Streaming

HLS + RTMP ingest, OBS-compatible. DVR support, adaptive bitrate, chat overlay, viewer analytics, and stream key management built in.

→
🔔
Push Notifications

FCM for Android, APNS for iOS, and SMTP email. Per-user preference management, quiet hours, and notification history built in.

→
🛡
Content Moderation

Auto-moderation rules (word/regex), bans with expiry, reports, and a full admin audit trail. Works across chat, streams, and calls.

→

Setup

Up and running in minutes

Three steps from zero to a fully operational real-time communication stack.

01
Deploy

Spin up all microservices, Kafka, Redis, and PostgreSQL with a single command. No complex configuration required to get started.

$ docker compose up -d
02
Integrate

Install any of our 7 official SDKs — Angular, React, React Native, Flutter, Android, iOS, or vanilla JS — and start using chat, calls, and streaming immediately.

npm i @rajutechie-streamkit/react-sdk
03
Scale

mediasoup workers scale horizontally, Kafka handles millions of events per minute, Redis manages presence for thousands of concurrent users.

kubectl scale deployment/signaling --replicas=8

SDK Code Showcase

Simple API. Powerful results.

Every feature is one service or hook away. Full TypeScript support across Angular, React, React Native, and more.

chat.component.ts
// Subscribe to real-time channel messages this.chatService .getChannelMessages('channel_general') .pipe(takeUntil(this.destroy$)) .subscribe(msg => { console.log(msg.text, 'from', msg.senderId); }); // Send a message with an attachment this.chatService .sendMessage('channel_general', { text: 'Hello, team! 👋', attachments: [{ mediaId: 'media_abc' }], }) .subscribe(); // React to a message this.chatService .addReaction('msg_xyz', '👍') .subscribe();
// Start an outgoing video call this.callService .startCall({ targetUserId: 'user_alice', type: 'video', }) .subscribe(call => console.log('Call ID:', call.id) ); // Handle incoming calls (real-time Observable) this.callService.incomingCalls$ .pipe(takeUntil(this.destroy$)) .subscribe(call => this.showIncomingCallUI(call) ); // Toggle audio/video mid-call this.callService.toggleAudio(callId).subscribe(); this.callService.toggleVideo(callId).subscribe();
// 1. Create a stream const res = await fetch('/streams', { method: 'POST', body: JSON.stringify({ title: 'My Live Stream', hostUserId: 'user_123', visibility: 'public', settings: { chatEnabled: true, maxBitrate: 6000 }, }), }).then(r => r.json()); // 2. Go live — returns HLS + RTMP URLs await fetch(`/streams/${res.data.id}/start`, { method: 'POST' }); // → rtmpUrl: rtmp://ingest.domain/live/<key> // → hlsUrl: https://cdn.domain/live/<id>/index.m3u8
// Update your presence status await fetch('/presence/user_123', { method: 'PUT', body: JSON.stringify({ status: 'online', // online|away|dnd|offline metadata: { device: 'desktop' }, }), }); // Bulk-lookup presence for a list of users const presence = await fetch('/presence/bulk', { method: 'POST', body: JSON.stringify({ userIds: ['user_1', 'user_2', 'user_3'], }), }).then(r => r.json()); // → [{ userId, status, lastSeen }, ...]
