# JavaScript SDK Guide

The RajutechieStreamKit JavaScript SDK (`@rajutechie-streamkit/js-sdk`) provides a complete client for building real-time chat, voice/video calls, meetings, and live streaming into browser applications.

## Installation

```bash
npm install @rajutechie-streamkit/js-sdk
# or
yarn add @rajutechie-streamkit/js-sdk
# or
pnpm add @rajutechie-streamkit/js-sdk
```

## Quick Start

```javascript
import { RajutechieStreamKit } from '@rajutechie-streamkit/js-sdk';

// Create a singleton client instance
const client = RajutechieStreamKit.getInstance({
  apiKey: 'sk_live_xxxxx',
  region: 'us-east-1',
});

// Connect with a user token (JWT generated by your server)
await client.connect(userToken);

// You are now connected and can use all modules
const channels = await client.chat.listChannels();
console.log('Channels:', channels.data);
```

## Configuration

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `apiKey` | `string` | **required** | Your RajutechieStreamKit project API key |
| `region` | `string` | `'us-east-1'` | Deployment region |
| `apiUrl` | `string` | `'https://api.rajutechie-streamkit.io/v1'` | REST API base URL |
| `wsUrl` | `string` | `'wss://ws.rajutechie-streamkit.io'` | WebSocket server URL |
| `logLevel` | `LogLevel` | `LogLevel.WARN` | Logging verbosity (`DEBUG`, `INFO`, `WARN`, `ERROR`) |
| `autoReconnect` | `boolean` | `true` | Automatically reconnect on connection loss |

## Chat Module

Access the chat module via `client.chat`.

### Channels

```javascript
// Create a channel
const channel = await client.chat.createChannel({
  type: 'group',
  name: 'Engineering Team',
  members: ['user-1', 'user-2', 'user-3'],
});

// List channels (with pagination)
const channels = await client.chat.listChannels({
  type: 'group',
  limit: 20,
});

// Get a specific channel
const channel = await client.chat.getChannel('channel-id');

// Update a channel
const updated = await client.chat.updateChannel('channel-id', {
  name: 'New Channel Name',
});

// Delete a channel
await client.chat.deleteChannel('channel-id');
```

### Members

```javascript
// Add members to a channel
const members = await client.chat.addMembers('channel-id', [
  { userId: 'user-4', role: 'member' },
  { userId: 'user-5', role: 'moderator' },
]);

// Remove a member
await client.chat.removeMember('channel-id', 'user-4');

// Update a member's role
const member = await client.chat.updateMember('channel-id', 'user-5', {
  role: 'admin',
});

// List members
const memberList = await client.chat.listMembers('channel-id', {
  limit: 50,
});
```

### Messages

```javascript
// Send a message
const message = await client.chat.sendMessage('channel-id', {
  text: 'Hello, world!',
  attachments: [
    { type: 'image', url: 'https://example.com/photo.jpg' },
  ],
});

// Get messages (paginated, newest first)
const messages = await client.chat.getMessages('channel-id', {
  limit: 25,
});

// Get a single message
const msg = await client.chat.getMessage('channel-id', 'message-id');

// Edit a message
const edited = await client.chat.editMessage('channel-id', 'message-id', {
  text: 'Updated message content',
});

// Delete a message
await client.chat.deleteMessage('channel-id', 'message-id');
```

### Reactions

```javascript
// Add a reaction to a message
await client.chat.addReaction('channel-id', 'message-id', '👍');

// Remove a reaction
await client.chat.removeReaction('channel-id', 'message-id', '👍');
```

### Read Receipts

```javascript
// Mark a message as read
await client.chat.markAsRead('channel-id', 'message-id');
```

### Typing Indicators

```javascript
// Signal that the current user started typing
client.chat.startTyping('channel-id');

// Signal that the current user stopped typing
client.chat.stopTyping('channel-id');
```

### Search

```javascript
// Search messages in a channel
const results = await client.chat.searchMessages('channel-id', {
  query: 'deployment schedule',
  senderId: 'user-1',           // optional: filter by sender
  hasAttachment: true,           // optional: only messages with attachments
  limit: 10,
});
```

### Channel Subscriptions

```javascript
// Subscribe to real-time events for a channel
client.chat.subscribe('channel-id');

// Unsubscribe when no longer needed
client.chat.unsubscribe('channel-id');
```

### Chat Event Listeners

```javascript
// New message received
client.on('message.new', (message) => {
  console.log(`New message in ${message.channelId}:`, message.content.text);
});

// Message was edited
client.on('message.updated', (message) => {
  console.log(`Message ${message.id} updated`);
});

// Message was deleted
client.on('message.deleted', ({ channelId, messageId }) => {
  console.log(`Message ${messageId} deleted from ${channelId}`);
});

// Typing indicators
client.on('typing.start', ({ channelId, userId }) => {
  console.log(`${userId} is typing in ${channelId}`);
});

client.on('typing.stop', ({ channelId, userId }) => {
  console.log(`${userId} stopped typing in ${channelId}`);
});
```

## Call Module

Access the call module via `client.call`.

### Making and Receiving Calls

```javascript
// Start a new call
const call = await client.call.start({
  type: 'video',               // 'audio' or 'video'
  participants: ['user-2'],
});

// Accept an incoming call
await client.call.accept('call-id');

// Reject an incoming call
await client.call.reject('call-id', 'Busy right now');

// End a call
await client.call.end('call-id');

// Get call details
const callInfo = await client.call.get('call-id');
```

### Media Controls

```javascript
// Toggle audio (mute/unmute)
client.call.toggleAudio('call-id', false);  // mute
client.call.toggleAudio('call-id', true);   // unmute

// Toggle video
client.call.toggleVideo('call-id', false);  // camera off
client.call.toggleVideo('call-id', true);   // camera on

// Switch camera (front/back on mobile)
client.call.switchCamera('call-id');

// Screen sharing
client.call.startScreenShare('call-id');
client.call.stopScreenShare('call-id');
```

### Recording

```javascript
// Start recording a call
await client.call.startRecording('call-id');

// Stop recording
await client.call.stopRecording('call-id');

// Get call statistics
const stats = await client.call.getStats('call-id');
console.log('Call duration:', stats.duration);
```

### Call Event Listeners

```javascript
// Incoming call
client.on('call.incoming', (call) => {
  console.log(`Incoming ${call.type} call from ${call.hostId}`);
});

// Call accepted
client.on('call.accepted', ({ callId, userId }) => {
  console.log(`${userId} accepted call ${callId}`);
});

// Call rejected
client.on('call.rejected', ({ callId, userId, reason }) => {
  console.log(`${userId} rejected call: ${reason}`);
});

// Call ended
client.on('call.ended', ({ callId, reason }) => {
  console.log(`Call ${callId} ended: ${reason}`);
});

// Participant joined
client.on('call.participant.joined', (participant) => {
  console.log(`${participant.userId} joined the call`);
});

// Participant left
client.on('call.participant.left', ({ callId, userId }) => {
  console.log(`${userId} left the call`);
});

// Recording events
client.on('call.recording.started', ({ callId }) => {
  console.log('Recording started');
});

client.on('call.recording.stopped', ({ callId }) => {
  console.log('Recording stopped');
});
```

## Meeting Module

Access the meeting module via `client.meeting`.

### Scheduling and Managing Meetings

```javascript
// Schedule a meeting
const meeting = await client.meeting.schedule({
  title: 'Sprint Planning',
  scheduledAt: '2026-03-01T10:00:00Z',
  duration: 60,  // minutes
  participants: ['user-1', 'user-2', 'user-3'],
  settings: {
    muteOnJoin: true,
    waitingRoom: true,
  },
});

// Get meeting details
const meetingInfo = await client.meeting.get('meeting-id');

// Update a meeting
const updated = await client.meeting.update('meeting-id', {
  title: 'Updated Sprint Planning',
});

// Cancel a meeting
await client.meeting.cancel('meeting-id');
```

### Joining and Leaving Meetings

```javascript
// Join a meeting
const participant = await client.meeting.join('meeting-id');

// Join by shareable code
const result = await client.meeting.joinByCode('ABC-DEF-GHI');
console.log('Joined meeting:', result.title);

// Leave a meeting
await client.meeting.leave('meeting-id');

// End a meeting (host only)
await client.meeting.end('meeting-id');
```

### Participant Management

```javascript
// Mute a participant (host only)
await client.meeting.muteParticipant('meeting-id', 'user-id');

// Remove a participant (host only)
await client.meeting.removeParticipant('meeting-id', 'user-id');

// Mute all participants (host only)
await client.meeting.muteAll('meeting-id');
```

### Hand Raise

```javascript
// Raise hand
client.meeting.raiseHand('meeting-id');

// Lower hand
client.meeting.lowerHand('meeting-id');
```

### Polls

```javascript
// Create a poll
const poll = await client.meeting.createPoll('meeting-id', {
  question: 'Which date works best?',
  options: [
    { id: 'opt-1', text: 'Monday' },
    { id: 'opt-2', text: 'Wednesday' },
    { id: 'opt-3', text: 'Friday' },
  ],
  anonymous: false,
  multipleChoice: false,
});

// Vote on a poll
await client.meeting.votePoll('meeting-id', 'poll-id', 'opt-2');
```

### Breakout Rooms

```javascript
// Create breakout rooms
const rooms = await client.meeting.createBreakoutRooms('meeting-id', {
  count: 3,
  assignAutomatically: true,
});
```

### Meeting Event Listeners

## Live Stream Module

Access the live stream module via `client.stream`.

### Creating and Managing Streams

```javascript
// Create a live stream
const stream = await client.stream.create({
  title: 'Product Launch Event',
  visibility: 'public',          // 'public' | 'private' | 'unlisted'
  settings: {
    lowLatency: true,
    chatEnabled: true,
    dvrEnabled: false,
    moderationEnabled: true,

  },
});

console.log('RTMP URL:', stream.rtmpUrl);

// Get stream details
const info = await client.stream.get('stream-id');

// Start the stream
const live = await client.stream.start('stream-id');
console.log('HLS URL:', live.hlsUrl);

// Stop the stream
await client.stream.stop('stream-id');

// Get viewer count
const { count } = await client.stream.getViewerCount('stream-id');
console.log('Current viewers:', count);
```

### Moderation

```javascript
// Moderate a stream (e.g., ban a user from the stream chat)
```

## Connection Management

### Connection Lifecycle

```javascript
const client = RajutechieStreamKit.getInstance({ apiKey: 'sk_live_xxxxx' });

// Connect with a user token
await client.connect(userToken);

// Check connection state
console.log('State:', client.connectionState);  // 'connected'
console.log('Connected:', client.isConnected);   // true

// Disconnect
await client.disconnect();
```

### Connection State Events

The connection state is one of: `'connecting'`, `'connected'`, `'disconnected'`, `'reconnecting'`.

```javascript
client.on('connection.changed', ({ state }) => {
  switch (state) {
    case 'connecting':
      console.log('Connecting to RajutechieStreamKit...');
      break;
    case 'connected':
      console.log('Connected!');
      break;
    case 'disconnected':
      console.log('Disconnected from RajutechieStreamKit');
      break;
    case 'reconnecting':
      console.log('Connection lost, attempting to reconnect...');
      break;
  }
});

client.on('connection.error', (error) => {
  console.error('Connection error:', error.message);
});
```

### Token Refresh

When a token is about to expire, generate a new one on your server and reconnect:

```javascript
// Listen for token expiration warning
client.on('connection.token_expiring', async () => {
  const newToken = await fetchNewTokenFromYourServer();
  await client.connect(newToken);
});
```

### Auto-Reconnection

When `autoReconnect` is enabled (default), the SDK automatically reconnects with exponential backoff when the WebSocket connection drops. The reconnection behavior follows a retry policy with configurable delays.

## Error Handling

All SDK errors extend the base `RajutechieStreamKitError` class and carry a machine-readable `code` property.

### Error Hierarchy

| Error Class | Description |
|-------------|-------------|
| `RajutechieStreamKitError` | Base error class for all RajutechieStreamKit errors |
| `AuthError` | Authentication and authorization errors |
| `NetworkError` | Network-level errors (offline, timeout, WebSocket) |
| `ValidationError` | Client-side validation errors |
| `ApiError` | HTTP 4xx/5xx errors from the RajutechieStreamKit API |

### Error Codes

```javascript
import { RajutechieStreamKitError, AuthError, NetworkError, ApiError, RajutechieStreamKitErrorCode } from '@rajutechie-streamkit/js-sdk';

try {
  await client.chat.sendMessage('channel-id', { text: 'Hello!' });
} catch (error) {
  if (error instanceof AuthError) {
    switch (error.code) {
      case RajutechieStreamKitErrorCode.AUTH_TOKEN_EXPIRED:
        // Refresh the token and retry
        break;
      case RajutechieStreamKitErrorCode.AUTH_TOKEN_INVALID:
        // Redirect to login
        break;
      case RajutechieStreamKitErrorCode.AUTH_UNAUTHORIZED:
        // User lacks permission
        break;
    }
  }

  if (error instanceof NetworkError) {
    if (error.retryable) {
      // Safe to retry this operation
      console.log('Network error, will retry:', error.message);
    }
  }

  if (error instanceof ApiError) {
    console.log('HTTP status:', error.statusCode);
    console.log('Response body:', error.responseBody);

    if (error.code === RajutechieStreamKitErrorCode.API_RATE_LIMITED) {
      // Back off and retry later
    }
  }

  if (error instanceof RajutechieStreamKitError) {
    // All errors have these properties
    console.log('Error code:', error.code);
    console.log('Timestamp:', error.timestamp);
    console.log('Metadata:', error.metadata);
    console.log('JSON:', JSON.stringify(error.toJSON()));
  }
}
```

### Common Error Codes

| Code | Category | Description |
|------|----------|-------------|
| `AUTH_TOKEN_EXPIRED` | Auth | JWT token has expired |
| `AUTH_TOKEN_INVALID` | Auth | JWT token is malformed or invalid |
| `AUTH_UNAUTHORIZED` | Auth | User lacks permission for this action |
| `NETWORK_OFFLINE` | Network | Device is offline |
| `NETWORK_TIMEOUT` | Network | Request timed out |
| `NETWORK_WEBSOCKET_ERROR` | Network | WebSocket connection error |
| `API_BAD_REQUEST` | API | Invalid request parameters (HTTP 400) |
| `API_NOT_FOUND` | API | Resource not found (HTTP 404) |
| `API_RATE_LIMITED` | API | Rate limit exceeded (HTTP 429) |
| `CHANNEL_NOT_FOUND` | Module | Channel does not exist |
| `MESSAGE_TOO_LONG` | Module | Message exceeds maximum length |
| `CALL_ALREADY_ACTIVE` | Module | User already has an active call |
| `MEETING_FULL` | Module | Meeting has reached capacity |

## Device Management

The SDK includes a device manager for enumerating and managing media devices:

```javascript
// List available devices
const devices = await client.devices.enumerateDevices();
console.log('Cameras:', devices.videoInputs);
console.log('Microphones:', devices.audioInputs);
console.log('Speakers:', devices.audioOutputs);

// Get user media
const stream = await client.devices.getUserMedia({
  audio: true,
  video: { width: 1280, height: 720 },
});

// Get screen share stream
const screenStream = await client.devices.getDisplayMedia({
  video: true,
  audio: false,
});

// Stop all tracks on a stream
client.devices.stopTracks(stream);
```

The `RajutechieStreamKit` class is a convenience wrapper. You can access the underlying `RajutechieStreamKitClient` directly for the full API surface:

```javascript
const coreClient = client.getClient();

// Access signaling for low-level WebRTC negotiation
const signaling = client.signaling;
```

## Cleanup

When your application unmounts or the user logs out, disconnect and clean up:

```javascript
await client.disconnect();

// Optionally clear all singleton instances (useful in tests)
RajutechieStreamKit.clearInstances();
```

> **Note:** The `RajutechieStreamKit.getInstance()` method is keyed by `apiKey`. Calling it with the same API key always returns the same instance. Use `clearInstances()` to reset during testing.
