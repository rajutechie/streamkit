02 Quick Start

Get StreamKit running locally in under five minutes.

ℹ

Prerequisites: Node.js 20+, pnpm 8+, Docker Desktop

bash
# 1 — Clone the repository
git clone https://github.com/rajutechie/streamkit.git
cd streamkit

# 2 — Install all workspace dependencies
pnpm install

# 3 — Start infrastructure (Postgres, Redis, Kafka)
docker compose up -d

# 4 — Copy environment template and configure
cp .env.example .env
# Edit .env — set DB_PASSWORD, REDIS_URL, JWT_SECRET, etc.

# 5 — Run database migrations
pnpm db:migrate

# 6 — Start all services in development mode
pnpm dev

# Services will be available at:
#   API Gateway  → http://localhost:3000
#   WebSocket    → http://localhost:3001
#   Auth service → http://localhost:3010
✓

Run curl http://localhost:3000/health to verify the gateway is up. You should receive {"status":"ok"}.

03 Authentication

StreamKit uses JWT bearer tokens. Issue a token with your API credentials, then include it in every subsequent request as Authorization: Bearer <token>.

Issue a Token

  • POST/auth/tokenExchange API key+secret for a JWT
  • POST/auth/refreshRefresh an expiring token
  • POST/auth/revokeRevoke / invalidate a token
javascript
// Issue a JWT token
const response = await fetch('http://localhost:3000/auth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    apiKey:    'sk_dev_rajutechie-streamkit_001',
    apiSecret: 'dev_secret_001_change_me_in_production',
    userId:    'user_123',
    role:      'user',        // 'user' | 'admin' | 'moderator'
  }),
});
const { token, refreshToken, expiresIn } = await response.json();

// Refresh before expiry
const refreshed = await fetch('http://localhost:3000/auth/refresh', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ refreshToken }),
}).then(r => r.json());

// Revoke a token
await fetch('http://localhost:3000/auth/revoke', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${token}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ token }),
});
⚠

Never expose your apiSecret in client-side code. Issue tokens from your backend server and pass only the token to the client.

Token Response Shape

FieldTypeDescription
tokenstringJWT bearer token (use in Authorization header)
refreshTokenstringLong-lived token used to obtain new access tokens
expiresInnumberSeconds until the access token expires (default 3600)
userIdstringThe user ID embedded in the token
rolestringRole granted: user, admin, or moderator

04 Angular SDK Integration

The official Angular SDK wraps all StreamKit APIs into injectable Angular services with RxJS Observables.

bash
npm install @rajutechie-streamkit/angular-sdk

Module Setup

typescript — app.module.ts
import { NgModule } from '@angular/core';
import { RajutechieStreamKitModule } from '@rajutechie-streamkit/angular-sdk';

@NgModule({
  imports: [
    RajutechieStreamKitModule.forRoot({
      apiKey:  'sk_dev_rajutechie-streamkit_001',
      apiUrl:  'http://localhost:3000',
      wsUrl:   'http://localhost:3001',
    }),
  ],
})
export class AppModule {}

Chat Service

typescript — chat.component.ts
import { RajutechieStreamKitChatService } from '@rajutechie-streamkit/angular-sdk';

@Component({ /* ... */ })
export class ChatComponent {
  messages: any[] = [];

  constructor(private chatService: RajutechieStreamKitChatService) {}

  ngOnInit() {
    // Subscribe to real-time message stream
    this.chatService
      .messageStream('channel_abc')
      .subscribe(msg => this.messages.push(msg));
  }

  sendMessage() {
    this.chatService
      .sendMessage('channel_abc', { text: 'Hello!' })
      .subscribe();
  }
}

05 Chat API

Full-featured messaging with channels, threads, reactions, read receipts, and cursor-based pagination.

Channel Endpoints

  • POST/channelsCreate a new channel
  • GET/channels?type=publicList channels (filter by type)

Message Endpoints

  • POST/channels/:id/messagesSend a message
  • GET/channels/:id/messages?limit=25&after=cursorList messages (cursor pagination)
  • PATCH/channels/:id/messages/:midEdit a message
  • DELETE/channels/:id/messages/:midDelete a message
  • POST/channels/:id/messages/:mid/reactionsAdd a reaction
  • POST/channels/:id/messages/:mid/readMark message as read

Member Management

  • POST/channels/:id/members/:uidAdd member to channel
  • GET/channels/:id/membersList channel members
  • PATCH/channels/:id/members/:uidUpdate member role
  • DELETE/channels/:id/members/:uidRemove member
javascript
const API = 'http://localhost:3000';
const authHeaders = {
  Authorization: `Bearer ${token}`,
  'Content-Type': 'application/json',
};

// Create a channel
const channel = await fetch(`${API}/channels`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({ name: 'general', type: 'public' }),
}).then(r => r.json());

// Send a message with mentions
const msg = await fetch(`${API}/channels/${channelId}/messages`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    text:     'Hello World!',
    mentions: ['user_456'],
  }),
}).then(r => r.json());

// Paginate messages (cursor-based)
const { messages, nextCursor } = await fetch(
  `${API}/channels/${channelId}/messages?limit=25&after=${cursor}`,
  { headers: authHeaders }
).then(r => r.json());

// Add a reaction
await fetch(`${API}/channels/${channelId}/messages/${msgId}/reactions`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({ emoji: '👍' }),
});

06 WebSocket Real-time Events

StreamKit uses Socket.IO on port 3001. Authenticate on connect, then join channels to receive targeted events.

Message Events

  • EVENTmessage.newNew message in a joined channel
  • EVENTmessage.updatedMessage was edited
  • EVENTmessage.deletedMessage was deleted
  • EVENTtyping.startUser started typing
  • EVENTtyping.stopUser stopped typing

Call & Presence Events

  • EVENTcall.incomingIncoming call notification
  • EVENTcall.acceptedRemote participant accepted
  • EVENTcall.rejectedRemote participant rejected
  • EVENTcall.endedCall was terminated
  • EVENTpresence.updateA user's presence status changed
  • EVENTpresence.getRequest current presence state
javascript
import { io } from 'socket.io-client';

const socket = io('http://localhost:3001', {
  auth: { token },  // JWT from /auth/token
});

// Join a channel room
socket.on('connect', () => {
  socket.emit('channel:join', { channelId: 'channel_abc' });
});

// Handle new messages
socket.on('message.new', (msg) => {
  console.log('New message:', msg);
  appendMessageToUI(msg);
});

// Typing indicators
socket.on('typing.start', ({ userId, channelId }) => {
  showTypingIndicator(userId);
});
socket.on('typing.stop', ({ userId }) => {
  hideTypingIndicator(userId);
});

// Emit typing events
inputEl.addEventListener('input', () => {
  socket.emit('typing:start', { channelId: 'channel_abc' });
  clearTimeout(typingTimer);
  typingTimer = setTimeout(() =>
    socket.emit('typing:stop', { channelId: 'channel_abc' }), 1500);
});

// Presence updates
socket.on('presence.update', ({ userId, status }) => {
  updatePresenceUI(userId, status);
});

07 Video Calls

Initiate audio or video calls between users. Calls support recording, live stats, and WebRTC-based peer connections managed by the server.

  • POST/callsInitiate a call (type: audio|video)
  • POST/calls/:id/acceptAccept an incoming call
  • POST/calls/:id/rejectReject an incoming call
  • POST/calls/:id/endEnd an active call
  • POST/calls/:id/recording/startStart call recording
  • POST/calls/:id/recording/stopStop call recording
  • GET/calls/:id/statsGet call quality stats
javascript — REST
// Start a video call
const call = await fetch(`${API}/calls`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    type:         'video',
    participants: ['user_456', 'user_789'],
  }),
}).then(r => r.json());

console.log('Call ID:', call.id);  // use in further operations

// Accept via socket event then REST
socket.on('call.incoming', async (call) => {
  await fetch(`${API}/calls/${call.id}/accept`, {
    method: 'POST', headers: authHeaders,
  });
});
typescript — Angular SDK
// Start a call
this.callService
  .startCall({ type: 'video', participants: ['user_456'] })
  .subscribe(call => console.log('Call started:', call.id));

// Handle incoming calls
this.callService.incomingCalls$.subscribe(call => {
  if (confirm(`Incoming call from ${call.initiatedBy}?`)) {
    this.callService.acceptCall(call.id).subscribe();
  } else {
    this.callService.rejectCall(call.id).subscribe();
  }
});

08 Meetings

Schedule and manage multi-participant meetings with moderation controls, hand-raise, and polls.

  • POST/meetingsSchedule a new meeting
  • GET/meetings/:idGet meeting details
  • POST/meetings/:id/joinJoin a meeting
  • POST/meetings/:id/leaveLeave a meeting
  • POST/meetings/:id/endEnd the meeting (host only)
  • POST/meetings/:id/participants/:uid/muteMute a participant
  • POST/meetings/:id/hand/raiseRaise hand
  • POST/meetings/:id/hand/lowerLower hand
  • POST/meetings/:id/pollsCreate a poll
typescript — Angular SDK
// Schedule a meeting
this.meetingService.scheduleMeeting({
  title:       'Team Sync',
  scheduledAt: new Date('2026-03-01T10:00:00Z').toISOString(),
  duration:    60,           // minutes
}).subscribe(meeting => {
  console.log('Meeting ID:', meeting.id);
});

// Live participants stream
this.meetingService
  .participants$('meeting_abc')
  .subscribe(participants => {
    this.participantList = participants;
  });

// Raise hand
this.meetingService.raiseHand('meeting_abc').subscribe();

// Create a poll
this.meetingService.createPoll('meeting_abc', {
  question: 'Should we extend the sprint?',
  options:  ['Yes', 'No', 'Need more info'],
  duration: 120,  // seconds
}).subscribe();

09 Live Streaming

Create and broadcast live streams using HLS for playback and RTMP for ingestion (OBS, FFmpeg, etc.).

  • POST/streamsCreate a stream session
  • POST/streams/:id/startGo live — returns HLS + RTMP URLs
  • POST/streams/:id/stopEnd the stream
  • GET/streams/:id/viewersGet live viewer count
  • POST/streams/:id/moderateBan viewer or disable chat
javascript
// Step 1 — Create stream
const { data: stream } = await fetch(`${API}/streams`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    title:      'My Live Stream',
    hostUserId: 'user_123',
    visibility: 'public',
    settings: {
      chatEnabled: true,
      lowLatency:  true,
    },
  }),
}).then(r => r.json());

// Step 2 — Go live (returns broadcast URLs)
const { data: live } = await fetch(`${API}/streams/${stream.id}/start`, {
  method: 'POST', headers: authHeaders,
}).then(r => r.json());

console.log('HLS playback:', live.hlsUrl);
// → https://cdn.example.com/hls/stream_abc/index.m3u8
console.log('RTMP ingest:', live.rtmpUrl);
// → rtmp://your-server/live/stream_abc?key=secret

// Use live.rtmpUrl as the stream server in OBS / FFmpeg
// Use live.hlsUrl in video.js / hls.js for playback

// Viewer count
const { count } = await fetch(`${API}/streams/${stream.id}/viewers`,
  { headers: authHeaders }
).then(r => r.json());

10 Presence

Track and publish user presence states. Updates are broadcast in real-time via WebSocket to subscribed clients.

  • POST/presence/updateSet current user's presence status
  • GET/presence/:userIdGet a single user's presence
  • GET/presence/bulk?userIds=id1,id2Bulk presence lookup
javascript
// Set presence to online
await fetch(`${API}/presence/update`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    userId: 'user_123',
    appId:  'my-app',
    status: 'online',   // 'online' | 'away' | 'dnd' | 'offline'
    device: 'web',
  }),
});

// Get presence for a single user
const presence = await fetch(
  `${API}/presence/user_123`,
  { headers: authHeaders }
).then(r => r.json());
// → { userId: 'user_123', status: 'online', lastSeen: '2026-02-28T10:00:00Z' }

// Bulk lookup
const presences = await fetch(
  `${API}/presence/bulk?userIds=user_123,user_456,user_789`,
  { headers: authHeaders }
).then(r => r.json());
// → [{ userId, status, lastSeen }, ...]

11 Push Notifications

Send push, email, or in-app notifications. Manage per-user preferences to honour quiet hours and channel mutes.

  • POST/notifications/sendSend a notification (push|email|in_app)
  • GET/notifications/preferences/:userIdGet notification preferences
  • PATCH/notifications/preferences/:userIdUpdate preferences
javascript
// Send a push notification
await fetch(`${API}/notifications/send`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    userId: 'user_456',
    type:   'push',          // 'push' | 'email' | 'in_app'
    title:  'New Message',
    body:   'You have a new message from Alice',
    data:   { channelId: 'channel_abc' },
  }),
});

// Update preferences — mute a channel
await fetch(`${API}/notifications/preferences/user_456`, {
  method: 'PATCH', headers: authHeaders,
  body: JSON.stringify({
    mutedChannels: ['channel_abc'],
    quietHours:    { start: '22:00', end: '08:00', timezone: 'Asia/Kolkata' },
  }),
});

15 Media Upload

Initiate an upload to get a pre-signed S3 URL, PUT the file directly to S3, then use the mediaId in messages or profiles.

  • POST/media/uploadInitiate upload — returns presigned PUT URL
  • GET/media/:idGet media metadata + fresh download URL
  • DELETE/media/:idDelete a media object
javascript
// Step 1 — Request a presigned upload URL
const { mediaId, uploadUrl } = await fetch(`${API}/media/upload`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    filename: 'photo.jpg',
    mimeType: 'image/jpeg',
    size:     204800,   // bytes
  }),
}).then(r => r.json());

// Step 2 — Upload directly to S3 (no auth header needed)
await fetch(uploadUrl, {
  method:  'PUT',
  headers: { 'Content-Type': 'image/jpeg' },
  body:    fileBlob,
});

// Step 3 — Use mediaId in a message
await fetch(`${API}/channels/${channelId}/messages`, {
  method: 'POST', headers: authHeaders,
  body: JSON.stringify({
    text:        'Check out this photo!',
    attachments: [{ mediaId, type: 'image' }],
  }),
});

// Get a fresh download URL
const { downloadUrl } = await fetch(
  `${API}/media/${mediaId}`,
  { headers: authHeaders }
).then(r => r.json());

17 Self-Hosting Guide

Deploy the full stack with Docker Compose. For production use Kubernetes manifests in /infrastructure/k8s.

Docker Compose — Infrastructure

yaml — docker-compose.yml
version: '3.9'
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB:       streamkit
      POSTGRES_USER:     streamkit
      POSTGRES_PASSWORD: '${DB_PASSWORD}'
    volumes:
      - pg_data:/var/lib/postgresql/data
    ports: ['5432:5432']
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U streamkit']
      interval: 10s
      retries: 5

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass '${REDIS_PASSWORD}'
    volumes:
      - redis_data:/data
    ports: ['6379:6379']

  zookeeper:
    image: confluentinc/cp-zookeeper:7.5.0
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181

  kafka:
    image: confluentinc/cp-kafka:7.5.0
    depends_on: [zookeeper]
    ports: ['9092:9092']
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_AUTO_CREATE_TOPICS_ENABLE: 'true'
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

volumes:
  pg_data:
  redis_data:

Nginx Reverse Proxy

nginx — /etc/nginx/conf.d/streamkit.conf
upstream api_gateway {
    server 127.0.0.1:3000;
}

upstream ws_gateway {
    server 127.0.0.1:3001;
}

server {
    listen 80;
    server_name streamkit.yourdomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name streamkit.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/streamkit.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/streamkit.yourdomain.com/privkey.pem;

    # REST API
    location /api/ {
        proxy_pass         http://api_gateway/;
        proxy_set_header   Host             $host;
        proxy_set_header   X-Real-IP        $remote_addr;
        proxy_set_header   X-Forwarded-For  $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto https;
        client_max_body_size 50m;
    }

    # WebSocket (Socket.IO)
    location /ws/ {
        proxy_pass         http://ws_gateway/;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade    $http_upgrade;
        proxy_set_header   Connection "upgrade";
        proxy_set_header   Host       $host;
        proxy_read_timeout 86400s;
    }
}
✓

For production deployments, use certbot --nginx to obtain free TLS certificates from Let's Encrypt, and configure CORS_ORIGIN in your .env to match your frontend domain.

⚠

Always rotate JWT_SECRET, database passwords, and AWS credentials before go-live. The default dev_secret_001_change_me_in_production value is for local development only.

usePresence Hook

Subscribes to real-time presence updates for a set of user IDs. Performs an initial bulk fetch then keeps the map live via WebSocket events.

tsx
import { usePresence } from '@rajutechie-streamkit/react-sdk';

function OnlineIndicator({ userId }: { userId: string }) {
  // Pass an array of IDs — hook re-subscribes only when the set changes
  const presenceMap = usePresence([userId]);
  const status = presenceMap.get(userId);

  return (
    <span className={status?.status === 'online' ? 'online' : 'offline'}>
      {status?.status ?? 'offline'}
    </span>
  );
}

Registering for Push Notifications

Call registerForPushNotifications after the user is authenticated. Pass a getToken function from your push library — the SDK stays decoupled from any specific provider.

tsx
import { registerForPushNotifications, handleNotification, dispatchNotification }
  from '@rajutechie-streamkit/react-native-sdk';
import messaging from '@react-native-firebase/messaging';

// 1. Register device — call once after login
await registerForPushNotifications(
  'fcm',          // 'fcm' | 'apns'
  streamKitClient, // authenticated RajutechieStreamKitClient
  () => messaging().getToken(),
);

// 2. Register a handler for incoming notifications
const unsub = handleNotification((payload) => {
  console.log('Notification:', payload.title, payload.body);
  // Navigate or update UI based on payload.type
});

// 3. Wire up your push library to dispatch to all handlers
messaging().onMessage(async (remoteMessage) => {
  dispatchNotification({
    title: remoteMessage.notification?.title,
    body:  remoteMessage.notification?.body,
    data:  remoteMessage.data,
    type:  remoteMessage.data?.type as string,
  });
});

// 4. Cleanup on logout
unsub();
clearNotificationHandlers();
FunctionSignatureDescription
registerForPushNotifications(provider, client, getToken) → Promise<string>Obtains a token, registers device with StreamKit backend
handleNotification(handler) → () => voidRegister a notification handler; returns cleanup fn
dispatchNotification(payload) → voidFan-out to all registered handlers
getRegisteredToken() → string | nullReturns the last registered push token
clearNotificationHandlers() → voidRemove all handlers (use on logout)
✓

For APNs (iOS), pass 'apns' as the provider and use messaging().getAPNSToken() as the getToken function. StreamKit auto-detects Platform.OS to set the device platform field.

Spring Boot — Token Service

java — TokenService.java
// Inject via @Service and call from AuthController
public String generateToken(String userId, String displayName, String role) {
    return Jwts.builder()
        .subject(userId)
        .claim("name", displayName)
        .claim("role", role)
        .issuer(apiKey)
        .expiration(Date.from(Instant.now().plus(24, ChronoUnit.HOURS)))
        .signWith(Keys.hmacShaKeyFor(apiSecret.getBytes()))
        .compact();
}
